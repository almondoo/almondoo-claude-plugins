# iterative-refine

セッション内で「実装 → 検証 → 修正 → 再検証」を反復するためのフレームワーク skill。終了条件(成功 / 上限到達 / 振動 / 検証回避)を明示することで、早すぎる打ち切り・スコープドリフト・検証ごまかし(テスト skip / アサーション緩和 / 検証コマンドの書き換え)を防ぎます。

## なぜ必要か

「収束するまで直して」と頼むだけでも Claude は動きますが、判断基準が無いと:

- 「だいたい良さそう」で止まる(早すぎる打ち切り)
- 行き詰まると test を skip / 削除して「pass した」と報告(検証ごまかし)
- 同じ箇所を A→B→A→B と往復(振動)
- 関係ないファイルを「ついでに」直す(スコープドリフト)

これらは裁量に任せる限り起き続けます。本 skill は **裁量を消すための明示的な枠組み**(終了条件 + 禁止行為 + 出力フォーマット)を提供します。

## いつ使うか

- 「全テストが pass するまで直して」
- 「型エラー / lint エラーを全部潰して」
- 「`<検証コマンド>` が exit 0 になるまでイテレーション」

逆に、`/loop` (ScheduleWakeup ベース) が必要なのは **セッションを跨いで再起動する** 場合(例: 30 分後に CI 結果を確認)のみ。同じ会話内のセルフリファインメントには本 skill を使ってください。

## 必要な入力

skill 起動時に以下が揃っていないと、`AskUserQuestion` で確認します。

1. **ゴール** — 1 文(例: 「`pnpm test` が全 pass する」)
2. **検証コマンド** — exit code で成否が決まる shell コマンド
3. **最大反復回数** — 既定 5
4. **スコープ** — 触ってよい / 禁止のファイル / glob

## 終了条件

| Code | 発火条件 |
|---|---|
| `success` | 検証コマンドが exit 0 |
| `escape-attempt` | テスト skip / アサーション緩和 / 検証コマンド書き換え / snapshot 上書き / broad ignore を実行しようとした |
| `oscillation` | 同じ検証出力が 2 連続(進捗なし) |
| `max-iterations` | 上限到達 |

`escape-attempt` は最優先。「もう 1 反復で抜けるため」だとしても、検証を緩める方向の行動は実行せずに停止して報告します。

## インストール

```
/plugin install iterative-refine@almondoo-claude-plugins
```
